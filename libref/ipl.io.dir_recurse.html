dir_recurse(f, on_err)

Package :
ipl.io
Type :
procedure

Recursively traverse directory f, generating all the filenames (including directories). If this generator isn't used to exhaustion, the call must be wrapped in the util.clean() function, in order to close all outstanding files. on_err is an optional procedure to invoke on an error reading a directory. If it throws an exception, then the call to dir_recurse should also be enclosed with one of the try procedures.

Example :- 

    clean{try1{{
       every f := dir_recurse(d, dir_recurse_throw) \ 50 do
          write("got ", f)
    }}} | write("Problem: ", &why)
See also :
