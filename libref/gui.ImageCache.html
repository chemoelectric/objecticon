get(s, win)

public static

Load an image, using the cache if possible. The parameter s should be either a file name, or the name of an image set by name_image(). On a successful load, the image is stored in the cache for future use, and thus should not be closed after use.

In this and subsequent methods, win is a Window. This parameter is only required if the image data in question is an image string. The window is then used to provide the foreground and background palette colours.

name_image(name, data)

public static

Associate image data (eg gif file data), with a name. The data must be either a string, or a Pixels instance. In the latter case, the instance must never be closed.

The name can then be passed to get or load to retrieve a window with the corresponding image. It is a runtime error if the key already exists in the table.
