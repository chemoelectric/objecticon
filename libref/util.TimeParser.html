Method detail

parse(s, p)

public

Parse the instance using the given pattern string. The pattern consists of pattern chars and other chars. Succeeds if the string could be parsed; fails and sets &why otherwise.

The "width" of a field is the number of successive equal pattern chars. For example in the pattern

yyyy/MMM/dd

the widths are 4, 3 and 2 respectively.

Except for the month and year (see below), the only use of the width is to separate adjacent numeric fields. For example, with the format "HHmmss", because the field following the "HH" is numeric, the width of the hours field is limited to two digits, and in the input "123211" consumes only "12" (rather than the whole sequence of digits).

The possible pattern chars are :-

E - The weekday (eg Mon) - Match a full or abbreviated week- day name, ignoring case.

F - Like E, but using the local weekday names.

y - The year. If the width is 4, then the year must be explicitly 4 digits. Otherwise, if the year is less than 70 it is taken to be 20xx; if it is less than 100 it is taken to be 19xx, otherwise it is as given.

d - The day of the month

H - The hour in the day in the 24 hour format

h - The hour in the day in the 12 hour clock format

M - The month of the year. If the width is less than 3 then the numeric value is expected, otherwise a match of either full or abbreviated month name is expected, ignoring case.

N - Like M, but using the local month names.

m - The minute in the hour

s - The second in the minute

a/A - am or pm. Case is ignored. If pm, then the hour is adjusted accordingly in the result. Should only be used with "h".

b/B - Like a/A, but using the local am/pm names.

z - The timezone (eg UTC or +0400).

T - The timezone in RFC3339 format (eg Z or +04:00).

x - Matches the remaining input, and can be used to skip extraneous trailing input, which would otherwise cause an error.

space - Matches zero or more whitespace (space and tab).

Initial and trailing whitespace is always skipped.

A literal string may be matched by enclosing it in single quotes. For example ',' matches exactly one comma.

Any other character matches zero or more instances of that character. For example the comma in "E, d" would match an optional comma between the weekday and the month day.
