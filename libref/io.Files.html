bulk_close(gap, start)

public static native

This method can be used after a posix.System.fork and before a posix.System.exec to close unnecessary duplicated file descriptors. The algorithm used is not perfect, but should be effective in almost all cases. Starting at descriptor number start, descriptors are closed in sequence until gap descriptors in succession are found to be absent from the file descriptor table (in other words, attempting to close them returns an error).

Parameters :
gap - the gap (default 512)
start - the starting descriptor number (default 3)
