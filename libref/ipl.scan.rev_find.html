rev_find(s1, s2, i, j)

Package :
ipl.scan
Type :
procedure

Search backwards for s1 in s2[i:j]. For each match, the position just to the right of the match is generated.

If no parameters are defaulted, the sequence generated is equivalent to 

     find(s1, s2, i, j) + *s1
but in reverse order.

Defaults:

  • s2 - &subject
  • i - &pos if s is defaulted, otherwise 0
  • j - 1

Source - scan.icn
