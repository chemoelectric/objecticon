handle_in_chain(e)

private

This is used to forward an in chain event to Imgs and TextFlows in the point chain. The reason they don't simply listen themselves is that they would then process the event in the wrong order. Since the bottom components come last (ie the base TextFlow), when the mouse leaves a link in the top-level and moves to a link in another TextFlow (above the base one), the base TextFlow would process the event last, and would clear the UI message, overwriting the new message written by the upper level link. So this traverses the point chain backwards, which is the correct order for this case.
